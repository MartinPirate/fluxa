# Fluxa Roadmap

## Phase 0: Foundation

- define vision and MVP
- set up repository structure
- establish core package boundaries
- sketch first public APIs

## Phase 1: UI And Style Kernel

- define `FluxaNode` and composition primitives
- define utility-first styling model
- map styling into a runtime representation
- test the ergonomics with a small showcase

## Phase 2: Compose Integration

- bind Fluxa UI primitives to Compose
- map Fluxa styles into Compose modifiers
- establish preview flows
- build the first polished sample screens

## Phase 3: CLI And Scaffolding

- scaffold a Fluxa app
- generate screens and components
- install theme tokens and project structure
- add example-driven templates

## Phase 4: App Framework

- navigation model
- state conventions
- effect model
- data and caching story
- testing story
